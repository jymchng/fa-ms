# Objective
To make it easier to create new TypeScript project with many of the boilerplate codes already done.

# How to Use

## Step One
Remember to add the following folders:

1. `dev_notes`

Run `npm i`, then you will see `.husky` folder, move the `pre-commit` file into it.

## Step Two
Populate the `schema.prisma` file under the `<root>/prisma` directory.

## Step Three
Edit these `.<stmt>.<env>` files under the `<root>/secrets` directory.

## Step Four
Edit these `.env.<env>` files under the `<root>` directory.

## Step Five
Run `npx nestia setup` to start `nestia`.

# Additional Usage

# To inspect docker container 

Use `docker compose -f docker-compose-dev.yaml exec -it bot-test-dev sh`
